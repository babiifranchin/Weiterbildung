Attributselektoren

Attributselektoren wählen Elemente anhand des Vorhandenseins oder Wertes eines Attributs aus.

CSS kennt drei Arten von Attributselektoren

  1. Attributpräsenz: [name]
  2. Attributwert: [name=wert], [name~=wert], [name|=wert]
  3. Teilübereinstimmung: [name^=wert], [name$=wert], [name*=wert]

1. Attributpräsenz

[name]
Repräsentiert ein Element mit einem Attribut name, d. h. es werden Elemente selektiert, die das Attribut besitzen (unabhängig vom zugewiesenen Wert)

Beispiel:

HTML-Fragment

...
   <h1>Suchmaschinen</h1>

   <ul>
      <li>
         <a href="https://www.google.de" title="Google Suchmaschine">Google</a>
      </li>
      <li>
         <a href="https://www.bing.com" title="Bing Suchmaschine">Bing</a>
      </li>
      <li>
         <a href="https://www.startpage.com" title="Startpage Suchmaschine">Startpage</a>
      </li>
   </ul>
   <div>
      <a href="#">&uarr; zurück zum Seitenanfang &uarr;</a>
   </div>
...

CSS-Fragment

...
   a {
      text-decoration: none;
      color: blue;
   }
   a[title] {
   text-decoration: underline dotted tomato;
   background-color: lightgoldenrodyellow;
   color: green;
   }
...

2. Attributwert

[name=wert]
Repräsentiert ein Element mit einem Attribut name, welches exakt die Zeichenkette „wert“ als Attributwert enthält.
[name~=wert]
Repräsentiert ein Element mit einem Attribut name, welches die Zeichenkette „wert“ im Wert des Attributs enthält und diese Zeichenkette durch ein Leerzeichen (davor und/oder danach) vom restlichen Inhalt des Wertes abgetrennt ist. Dabei kann die Zeichenkette wie folgt vorkommen:
  • innerhalb des Attributwerts mit einen vorangehenden und nachfolgenden Leerzeichen
  • am Anfang des Attributwerts mit einem nachfolgenden Leerzeichen
  • am Ende des Attributwerts mit einem vorangehenden Leerzeichen
  • oder es entspricht exakt dem Attributwert
[name|=wert]
Repräsentiert ein Element mit einem Attribut name. Es kann exakt die Zeichenkette „wert“ als Attributwert enthalten oder mit der Zeichenkette „wert“ beginnen und mit eimen Bindestrich („-“ / U+002D) vom restlichen Inhalt des Wertes abgetrennt ist.
Der Attributwert muss dabei immer mit der Zeichenkette „wert“ beginnen.

Beispiel:

HTML-Fragment

...
   <!-- [name=wert] -->
   <h3>Der [name=wert]-Selektor</h3>
   <p><label for="name">Name: </label><input type="text" value="name" id="name"><br>
   <label for="email">E-Mail: </label><input type="email" value="email" id="email"><br>
   <input type="submit" value="Absenden"></p>
 
   <!-- [name~=wert]-->
   <h3>Der [name~=wert]-Selektor</h3>
   <ul>
     <li title="Berlin: eine schöne Stadt">Berlin: eine schöne Stadt</li>
     <li title="Berlin ist die Hauptstadt der BRD">Berlin ist die Hauptstadt der BRD</li>
     <li title="Die Stadt Berlin besteht aus mehreren Stadteilen">Die Stadt Berlin besteht aus mehreren Stadteilen</li>
     <li title="Das Brandenburger Tor steht in Berlin">Das Brandenburger Tor steht in Berlin</li>
     <li title="Aktuelle News und Meldungen aus Berlin.">Aktuelle News und Meldungen aus Berlin:</li>
     <li title="Berlin">Berlin</li>
     <li title="berlin">berlin</li>
   </ul>
   <!-- [name|=wert] -->
   <h3>Der [name|=wert]-Selektor</h3>
   <p lang="de">Hallo Welt</p>
   <p lang="en">Hello World</p>
   <p lang="de-ch">Grüezi Wäut</p>
   <p lang="en-de">Hallo World</p>
   <p lang="es">Holla Mundo</p>
...

CSS-Fragment

...
   [type=email] {outline: 3px solid tomato;}
   [title~=Berlin] {color: tomato;}
   [lang|=de] {color: tomato;}
...

3. Teilübereinstimmung

[name^=wert]
Repräsentiert ein Element mit einem Attribut name, dessen Wert mit „wert“ beginnt.
[name$=wert]
Repräsentiert ein Element mit einem Attribut name, dessen Wert mit „wert“ endet.
[name*=wert]
Repräsentiert ein Element mit einem Attribut name, dessen Wert mindestens einmal „wert“ enthält.

Beispiel:

HTML-Fragment

...
   <h1>Teilübereinstimmungen</h1>
   <ul>
      <li><a href="https://example.com/">Beispiellink</a> (fett)</li>
      <li><a href="https://example.com/example.pdf">Link zu einem PDF-Dokument</a> (fett grün)</li>
      <li><a href="https://wiki.selfhtml.org/">Link zum selfhtml wiki</a> (fett kursiv)</li>
      <li><a href="http://example.org/WIKI/example.pdf/">Kein Selektor darf diesen Link stylen.</a>
      </li>
   </ul>
...

CSS-Fragment

...
   a[href^="https://"] {
      font-weight: bold
   }
   
   a[href$=".pdf"] {
      color: green
   }
   
   a[href*="wiki"] {
      font-style: italic
   }
   ...



Teilübereinstimmungen

  • Beispiellink (fett)
  • Link zu einem PDF-Dokument (fett grün)
  • Link zum selfhtml wiki (fett kursiv)
  • Kein Selektor darf diesen Link stylen.

Anmerkungen - Case-insensitive modifier

Mittlerweile unterstützen die meisten Browser den sog. Case-insensitive modifier, mit dem man die Berücksichtigung von Groß-/Kleinschreibung deaktivieren kann.

[attr operator wert i]
Wenn ein i vor der schließenden Klammer eingefügt wird, wird ein Vergleich ohne Berücksichtigung der Groß-/Kleinschreibung (für Zeichen innerhalb des ASCII Bereichs) durchgeführt.

Beispiel:

HTML-Fragment

...
   <p><a href="http://example.org/WIKI/example.pdf/">Jetzt kann der Selektor diesen Link stylen.</a></p>
...

CSS-Fragment

...
   a[href*="wiki" i] {
      font-style: italic;
      font-weight: bold;
   }
...

Jetzt kann der Selektor diesen Link stylen.

Anmerkungen - default Attribute

Attributselektoren wirken i. R. nur auf Attribute, die im Quelltext geschrieben wurden. In HTML (und auch XML) gibt es Elemente die Attribute automatisch (per default) besitzen, deren Wert vordefiniert ist. Bei den meisten Browsern werden diese nicht selektiert.

Beispiel:

HTML-Fragment

...
   <h2>Nicht explizit angegebene Attribute</h2>
   <section>
      <p><label>Vorname</label> <input name="vorname"></p>
      <p><label>Nachname</label> <input type="text" name="nachname"></p>
   </section>
...

CSS-Fragment

...
   [type=text] {
        outline: 2px solid tomato;
   }
...
